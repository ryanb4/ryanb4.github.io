HASHMAP

  • Java

OVERVIEW

This is an implemenation of a basic Hash Map data structure that hashes strings into a table, ensures the keys are unique, and provides the user with O(1) time for access operations. This implemenation has faster access times than the TreeMap implemenation, but it cannot return all of the keys in sorted order without using an additional sort function. Therefore, the Hash Map will also provide a function to return all of the keys in in an unsorted array.

Also included are methods to alter the hash table, such as lambda and extend. The method lambda returns the lambda value for the table (the ratio bewtween the table size and the number of keys stored). Lambda is used to determine when the table size should be extended to avoid collisions. The mehtod extend thus doubles the size of the hash table and rehashes the keys using the new table size.

Chaining is used for collision management. When a key is hashed to the same location on the hash table as another key, the new key is linked to the end of the old key. Therefore, this type of collison management means that no key will be excluded from the hash table for being too full. However, the hash table will be extended when lambda becomes greater than or equal to one. This ensures that the access operations of the Hash Map are O(1).
