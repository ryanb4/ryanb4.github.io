OVERVIEW

This is an implementation of a cache using my own implemenation of a minimum binary heap and a hash map from the Java collections library. The replacement policy for the cache is to replace the least frequently used data. To implement this policy, each cache frame has a counter that is increased when a frame is referenced. Thus, when a frame needs to be replaced, the least frequently used data will be at the top of the minimum binary heap, so it will be removed.

The hashmap is used in our implemenation to help with searching through the cache frames. Instead of traversing the minimum binary heap to locate a cache frame in O(n) time, we can find cache frames already on the heap in O(1) time. This is important because we need to increase the counter on each cache frame every time it's referenced.
